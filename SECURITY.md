# Security Policy / Política de seguridad

## English

### Supported versions

This repository currently supports security fixes on the latest `main` branch.

### Reporting a vulnerability

- Do not open public issues for security problems.
- Email `security@example.com` with a clear description, impact, reproduction steps, and any proof of concept.
- We will acknowledge receipt as soon as possible and coordinate a fix before public disclosure.

### Privacy notes

SpecShowdown is local-first in this version. Setup data is stored in the browser unless a user exports it or shares a link manually.

## Español

### Versiones soportadas

En este repositorio se atienden correcciones de seguridad sobre la versión más reciente de la rama `main`.

### Cómo reportar una vulnerabilidad

- No abras issues públicos para problemas de seguridad.
- Envía un correo a `security@example.com` con descripción, impacto, pasos de reproducción y pruebas si existen.
- Confirmaremos recepción lo antes posible y coordinaremos una corrección antes de la divulgación pública.

### Notas de privacidad

En esta versión, SpecShowdown es local-first. Los datos de setups viven en el navegador salvo que la persona usuaria los exporte o comparta un enlace manualmente.
